vue-router

发表于 2020-07-27 | 分类于 前端

1、vue-router 是什么

vue-router 是Vue.js官方的路由插件，是单页应用（SPA）的路由管理器，它和Vue.js的核心深度集成的，适用于构建单页面应用。

Vue做的都是单页面应用，是基于路由和组件的，路由用于设定访问路径，并将路径和组件映射起来。

路由模块的本质是：建立起url和页面之间的映射关系。

2、vue-router 的实现原理

SPA(单一页面应用程序)，只有一个完整的页面，在加载页面的时候，不会加载整个页面，只是更新某个指定容器中的内容。

单页面应用的核心是：更新视图而不重新请求页面

vue-router 实现单页面前端路由提供两种方式：Hash模式和history模式，有参数mode来设定，默认采用Hash。

2.1、Hash

vue-router默认使用Hash模式，通过Hash 来模拟一个完整的url

hash(#)是一个url的锚点，代表的是网页中的一个位置，改变#后的部分浏览器之后滚动到相应位置，不会重新加载页面。

hash不会被包含在HTTP请求中，对后端完全没影响，因此完全不会被重新加载

每次改变#后边的部分，都会向浏览器的访问历史中加一个记录，，点击后退就可以的返回上一个位置。

所以说，hash模式通过锚点值的改变，根据不同的值，渲染指定DOM位置的不同数据。hash模式的原理是onhashchange 事件，可以在windo上监听。

2.2、history模式

如果不想让url中带有#，那可以使用history模式，只需要在配置路由规则时，添加“mode:history”即可。

这种模式充分利用了HTML5 中新增的pushSate()和replaceSate()方法。这两个方法应用于浏览器记录栈，在当前已有的back、forward、go的基础上，提供了对历史记录修改的功能，只是当他们执行修改时，虽然改变了当前的url但是浏览器不会立即向后端发送请求。

在使用history模式的时候，url就像是正常的url，比如 http://localhost:8081/id 。但是需要后端配置的支持。我们的应用是个单页的客户端应用，当用户直接在浏览器访问 http://localhost:8081/id 的时候，会返回404。所以在服务端加一个覆盖所有情况的候选资源：如果url匹配不到任何静态资源，就返回这个页面

8、$router 和 $route

$route 是：路由信息对象

  • path:当前路由的绝对路径
  • params:动态匹配和全匹配片段。
  • query：url查询参数
  • hash：当前路由hash值，不带#
  • fullPath:包含查询参数和hash值得完整路径
  • matched:数组，当前匹配路径中包含的所有片段所对应的配置参数对象
  • name:当前路径名字

$router 是：路由实例对象，是使用 new VueRouter() 创建的实例。

8.1、push和replace区别

  • push会向history栈里添加一个新的纪录，点击返回可以返回到之前页面
  • replace不会向history中添加新纪录，而是替换掉当前的history记录，replace跳转之后，后退按钮不能查看之前的页面。

9、404 页面

当用户输入错误页面时，我们希望给一个友好提示的页面，也就是404，vue-router为我们提供了这样一个机制。

10、响应路由参数变化

当使用路由参数时，比如从/user/foo到/user/bar，原来的组件实例会被复用，因为两个路由都会渲染同一个组件，比起销毁再创建，复用则显得非常有效。不过这意味着组件的生命周期钩子函数不会再被调用

比如说在博客的文章与文章之间，根据id不同进行跳转，组件会被复用，不会重新创建，那么created等钩子函数不会被调用

如果想对路由参数变化作出响应的话，可以通过watch监测$route变化，或者通过2.2的 beforeRouterUpdate 守卫

11、完整的 vue-router 导航解析流程

  • 1.导航被触发
  • 2.在失活的组件里调用离开守卫
  • 3.调用全局的beforeEach守卫
  • 4.在重用的组件里调用 beforeRouteUpdate 守卫（2.2+）
  • 5.在路由配置里调用beforeEnter
  • 6.解析异步路由组件
  • 7.在被激活的组件里边调用 beforeRouteEnter
  • 8.调用全局的 beforeResole 守卫（2.5+）
  • 9.导航被确认
  • 10.调用全局的 afterEach 钩子
  • 11.触发DOM更新
  • 12.用创建好的实例调用 beforeRouteEnter 守卫传递给next

导航

表示路由正在发生改变

守卫方法

每个守卫方法，接受三个参数：

  • to:Route 即将进入的目标路由对象
  • from:Route 当前导航正要离开的路由
  • next:Function 一定要调用该方法来resolve当前钩子，执行的效果依赖参数。
  1. next(): 进入下一个钩子，如果钩子全部执行完了，导航的状态就是confirm(确认的)
  2. next(false)：中断当前导航。如果url改变了，就退回from
  3. next(‘/‘) 或者 next({path:’/‘})：跳转到不同的网址。当前导航中断，开始新的导航。可以向next传递任何对象，允许设置任何router-link 和 router.push中的选项
  4. next(error)：（2.4.0+），导航终止，且该错误会被传进router.onError()注册回调函数。

要确保调用 next 否则钩子不会被 resolved

注意：参数或者查询的改变，不会触发进入/离开的导航守卫，可以通过watch $route 对象或者 beforeRouteUpdate 的组件守卫 来应对这些变化

12、几种实例方法及参数传递

12.1、this.$router.push(location,onComplete?,onAbort?)

向history栈里添加一个新的纪录，点击后退按钮时，回到之前的url。

并且用户点击 等同于调用 router.push(…) 。

12.2、this.$router.replace(locaton,onComplete?,onAbort?)

不会向 history 中添加记录，而是替换掉当前记录，所以当点击后退按钮时不会返回之前的 url 。

12.3、this.$router.go(n)

参数是一个整数，意思是在 history 记录中向前或者向后退多少步，类似 window.history.go(n)

在2.2.0+，可选的在 router.push 和 router.replace 中提供了onComplete 和 onAbort 回调作为第二第三个参数。这些回调将会在导航成功完成或者终止的时候进行相应调用。

14、router-link 组件及属性

to (string/location)

表示目标路由的链接。

当被点击后，内部会立刻把 to 值传到 router.push() 。

replace (boolean 默认false)

设置 replace 属性的话，当点击时，会调用 router.replace() 而不是 router.push() , 于是导航后不会留下 history 记录。

append (boolean 默认 false)

设置 append 属性后，则在当前（相对）路径前添加基本路径。

比如，我们从 /a 导航到一个相对路径 b ，如果没有配置 append ，则路径为 /b ，如果配置了，则为 /a/b
