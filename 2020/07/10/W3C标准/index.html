W3C标准

发表于 2020-07-10 | 分类于 前端

概念

W3C标准，万维网联盟（World Wide Web Consortium）标准，不是一个标准，而是一系列标准的集合。网页主要有三方面组成：

  • 结构
  • 表现
  • 行为

对应的标准也有三方面：

  • 结构化标准主要包括XHML和XML
  • 表现标准语言主要包括CSS
  • 行为标准主要包括W3C DOM 、ECMAScript等

这些标准大多数是由W3C起草发布的，也有一些是其他标准组织只能给定的，比如ECMAScript的ECMAScript标准。

XHTML

  • XHTML是更严谨纯净版的HTML
  • XHTML标准与HTML4几乎没有什么不同

主要的不同是：

  • XHTML元素必须正确被嵌套
  • XHTML元素必须被关闭
  • 标签名必须小写
  • XHTML文档必须有根元素

标准规范

1. 需要声明(DOCTYPE)

document type 文档类型的缩写，用来说明你用的XHTML或者HTML是什么版本。

DTD是文档类型定义，里面包含了文档规则，浏览器根据DTD来解释你的页面标识，并展现出来。

HTML1.0 和HTML4.01中都有三种DTD声明可以选择：过度的(Transitional)、严格的(Strict)、框架的(Frameset)。声明方式不同。

结论

1、标签规范可以提高搜索引擎对页面的抓取效率，对SEO（搜索引擎优化）很有帮助。

2、尽量使用外链css样式表和js脚本。是结构、表现和行为分为三块，符合规范。同时提高页面渲染速度，提高用户的体验。

3、样式尽量少用行间样式表，使结构与表现分离，标签的id和class等属性命名要做到见文知义，标签越少，加载越快，用户体验提高，代码维护简单，便于改版

注意：JQuery 不符合W3C标准
